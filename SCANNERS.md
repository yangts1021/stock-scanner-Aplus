# 掃描器規格與差異

本專案目前有兩個台股掃描器：`scan.py`（舊版基礎版）與 `today_scan_v7.py`（目前主用）。
記錄兩者篩選條件與差異以供日後對照。

## 1. `scan.py` 篩選條件

5 個條件全數通過才算命中：

| #   | 條件          | 規則                                                                                                                                                                |
| --- | ------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 1   | 長期多頭      | 收盤 > 60MA；60MA 20 日斜率 > 0                                                                                                                                     |
| 2   | 主升段曾啟動  | 過去 60 日內，**任何 20 日漲幅 ≥ 30%**                                                                                                                              |
| 3   | 高檔回檔      | 自 60 日高點回檔 **10%~40%**（註：原 docstring 寫 15~35%，實作為 10~40%）                                                                                           |
| 4   | 均線收斂      | 5MA / 10MA / 20MA 三者最大價差 ≤ 10%                                                                                                                                |
| 5   | 今日放量啟動  | • 漲幅 ≥ 3%<br>• 收盤 > 5MA 且 > 10MA，High 觸及 20MA<br>• 量 ≥ 5 日均量 × 1.5<br>• 今日 K 棒實體 > 過去 10 日平均實體                                               |

## 2. `scan.py`（純量 / 迴圈）vs `scan_vectorized.py`（向量化）

兩者骨架相同（5 條件 + 同方向），差異主要在程式風格、回檔閾值與重用性。

### 2.1 程式風格與運算範圍

| 面向             | `scan.py`（純量／for-loop）                                | `scan_vectorized.py`（向量化）                                       |
| ---------------- | ---------------------------------------------------------- | -------------------------------------------------------------------- |
| 計算對象         | Python for-loop 一格一格走（`iloc[j]`、`iloc[-1]`）        | 整條時間序列一次運算（pandas Series + `rolling`）                    |
| 輸出             | 「今天是否命中」單一 dict（`iloc[-1]` 當今天）             | **每一日**的 Boolean Series（True = 該日為訊號日）                   |
| 重用性           | 只算單一日，回測無法直接共用                               | 回測（`backtest_v7.py`）與當日掃描（`today_scan_v7.py`）**共用同一函式** |
| 速度             | 慢（每檔多重迴圈）                                         | 快（NumPy/pandas C 底層批次運算）                                    |
| 漲停 Close 修補  | 有（yfinance 收盤未寫入時用 High 補）                      | 無（直接吃 SQLite 收盤後資料，不需要修補）                           |
| 資料來源         | yfinance 直抓 + pickle cache（內建於本檔）                 | 函式只吃外部 DataFrame，不關心來源                                   |

### 2.2 5 條件逐條對應

| #   | 條件             | `scan.py`（純量寫法）                                                                | `scan_vectorized.py`（向量寫法）                                                                          |
| --- | ---------------- | ------------------------------------------------------------------------------------ | --------------------------------------------------------------------------------------------------------- |
| 1   | 長期多頭         | `c0 > ma60.iloc[-1]` 且手算 60MA 20 日斜率                                           | `(close > ma60) & (ma60.pct_change(20) > 0)`                                                              |
| 2   | 主升段曾啟動     | `for j in range(20, 60): max(gain)` 逐日比對                                         | `close.pct_change(20).rolling(60).max() ≥ 0.30`                                                           |
| 3   | 高檔回檔         | `(high60_max - c0) / high60_max ∈ [0.10, 0.40]` ← **peak → close**                  | `rolling_pullback(high, low, 60) ∈ [0.10, 0.30]` ← **peak → trough**（峰至谷最低 Low）                    |
| 4   | 均線收斂         | `(max(ma_vals) - min(ma_vals)) / min(ma_vals) ≤ 0.10`（單日三個 MA）                | `pd.concat([ma5,ma10,ma20], axis=1)` 算每日 spread `≤ 0.10`                                               |
| 5   | 今日放量啟動     | 同條件（漲幅 ≥3% / >MA5,MA10 / High≥MA20 / vol≥vol5×1.5 / body≥body10）—逐日 if 串接 | 同條件，但全部用布林 Series `&` 串：`(chg≥0.03) & (close>ma5) & (close>ma10) & (high≥ma20) & (vol≥vol5_prev×1.5) & (body≥body10×1.0)` |

### 2.3 閾值常數比較

| 常數             | `scan.py`        | `scan_vectorized.py` (`MIN_*` / `PULLBACK_RANGE`) |
| ---------------- | ---------------- | ------------------------------------------------- |
| MIN_BARS         | 80               | 80                                                |
| 60MA 斜率門檻    | > 0              | > 0                                               |
| 20 日漲幅門檻    | ≥ 30%            | ≥ 30%                                             |
| 回檔語意         | peak → close     | **peak → trough**（峰至谷）                       |
| 回檔範圍         | **10%~40%**      | **10%~30%**                                       |
| MA spread 上限   | 10%              | 10%                                               |
| 今日漲幅門檻     | ≥ 3%             | ≥ 3%                                              |
| 量比門檻         | × 1.5            | × 1.5                                             |
| 實體倍數         | × 1.0            | × 1.0                                             |

### 2.4 一致性驗證

`scan_vectorized.verify_against_v2(target_date)` 會跑全市場、把向量化版與舊純量版（`scan_v2.analyze`）在指定日的命中清單做集合差，確保兩種寫法輸出相同。

## 3. `scan.py` vs `today_scan_v7.py`（系統層差異）

| 項目         | `scan.py`（舊）                       | `today_scan_v7.py`                                  |
| ------------ | ------------------------------------- | --------------------------------------------------- |
| 資料來源     | yfinance 直抓 + pickle cache          | 本地 SQLite (`twstock.db`)                          |
| 形態邏輯     | 上述 5 條件硬篩（純量）               | `scan_vectorized.signals_for_ticker`（向量化）      |
| 回檔範圍     | 10%~40%                               | 10%~30%                                             |
| 大盤過濾     | 無                                    | TAIEX 收盤 > 60MA 且 60MA 20 日斜率 > 0             |
| R:R 計算     | 無                                    | 有（est_rr，依前高/止損推算）                       |
| 輸出         | 命中清單 + scan_results.csv           | 含 R:R 分桶 + Tier 標記、box-grid 表格              |

## 結論

- `scan.py` 為純量／單日掃描的基礎版，目前已不主用。
- `scan_vectorized.py` 是核心形態邏輯，由 `today_scan_v7.py`（掃描）與 `backtest_v7.py`（回測）共用，**保證掃描與回測同口徑**。
- 主要差異是「回檔範圍 10%~40% → 10%~30%」的口徑收窄，以及把寫法從 for-loop 改成向量化以共用與加速。
